Häufige Ursachen und Lösungen
Mehrere Python‑Installationen: Du hast z. B. System‑Python und eine andere Installation; pip installiert in einer anderen Version. Lösung: immer python -m pip verwenden oder den vollen Pfad zum Interpreter.

Virtualenv nicht aktiviert: Du hast Flask in .venv installiert, aber das Environment nicht aktiviert. Aktiviere es vor dem Start.

IDE‑Interpreter falsch gesetzt: Wenn du VS Code, PyCharm o. Ä. nutzt, stelle sicher, dass das Projekt‑Interpreter auf das Environment zeigt, in dem Flask installiert ist.

Rechte/Proxy: Bei Firmen‑PCs kann die Installation durch Proxy/Firewall blockiert werden. Prüfe Fehlermeldungen beim pip install.

Wenn du Docker oder Production nutzt
In Dockerfile: füge RUN pip install Flask oder COPY requirements.txt + RUN pip install -r requirements.txt.

In Production: verwende ein dediziertes virtualenv oder Systemd‑Service, das den richtigen Interpreter nutzt.
